# Integration: Slack

## What it does

Real-time communication and the place most decisions *start* as a conversation — but should not be where they *end*. Slack threads scroll away; this repo doesn't.

## Common workflows

- **Pull a Slack thread into `commands/analyze-feedback.md`** when feedback or signal accumulates in a channel — paste the thread as raw input rather than summarizing from memory.
- **After a decision is reached in a thread, write it down** in `artifacts/decision-log-template.md` and link back to the thread for full context. The log entry is the source of truth; the thread is supporting evidence.
- **Use `commands/write-exec-summary.md` before posting a status update** to a leadership channel, so the update leads with the decision/ask instead of a narrative.
