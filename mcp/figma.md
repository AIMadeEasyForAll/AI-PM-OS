# Integration: Figma

## What it does

Holds the design artifacts — wireframes, prototypes, the visual exploration that goes alongside a PRD. This repo holds the written scope; Figma holds what it looks like.

## Access

- Team: [add your Figma team URL here]
- Required access: view access at minimum to reference designs in artifacts; comment access to leave PM feedback in context.

## Common workflows

- **Link the relevant Figma frame directly in the PRD** (`artifacts/PRD-template.md`) once design work starts, rather than describing the UI in prose.
- **Reference Figma links in `projects/your-project/`** so the project folder is a complete record — scope, design, and outcome in one place even though the design file itself lives elsewhere.
- **Use the user-flow diagram** (`artifacts/diagrams/user-flow.drawio`) as the PM-level flow before design starts; Figma takes over for pixel-level flow once scope is settled.
