# Glossary

Shared definitions so "MVP" or "north star metric" means the same thing every time this system is used. Edit these to match how your org actually uses each term — the point is consistency, not textbook correctness.

| Term | Definition |
|---|---|
| **MVP** | The smallest version of a feature that tests the problem statement. Not "the full feature with fewer polish passes." |
| **North star metric** | The single metric the product organization is most accountable to moving, e.g. weekly active accounts retained at 90 days. |
| **Leading indicator** | A metric that moves fast and is closely attributable to a specific change — used for shipping decisions. |
| **Lagging indicator** | A metric that moves slowly and reflects many compounding factors — used for quarterly/strategic review. |
| **JTBD (Jobs-to-be-Done)** | The functional, emotional, and social job a user is "hiring" a product or feature to do. |
| **Guardrail metric** | A metric that shouldn't get worse as a side effect of a change, even if it isn't the primary success measure. |
| **Definition of done** | The explicit, agreed bar — usually metric-based — that determines whether a feature shipped successfully, set before launch. |
| **Discovery** | Validating that a problem is real and worth solving, before any scoping or building happens. |
| **Eval (evaluation)** | A structured, repeatable test of whether an AI/model-powered feature produces acceptable output, against a defined test set and threshold. |
| **RICE** | Reach × Impact × Confidence ÷ Effort — a relative prioritization score, not an exact ranking. |
