# Command: Write Exec Summary

Reusable prompt for turning a decision, launch, or project update into a leadership-ready summary, using the stakeholder-management skill.

## Prompt

```
Act as a senior product manager. Use the stakeholder-management skill (skills/stakeholder-management.md).

Using the provided update, write an executive summary structured as:
1. Decision or ask — 1-2 sentences, stated first, no preamble
2. Context — 3-5 sentences maximum, only what's needed to support the decision
3. What's needed from the reader, if anything (a decision, a sign-off, awareness only)
4. Where to follow up with questions

Translate the update to what this audience cares about (revenue, customer impact, risk, or their own roadmap — check settings/personas.md and settings/business-context.md for who this is and what they track). Do not bury bad news — if the update is a miss or a delay, say so in sentence one.

Audience: [WHO IS READING THIS]
Update: [PASTE THE RAW UPDATE, DECISION, OR PROJECT STATUS HERE]
```

## When to use this

Status updates, launch announcements, or any time a decision needs to travel to people who weren't in the room.

## Inputs to paste in

- The raw update or decision, unpolished
- Who's reading it
- Any prior commitment this update confirms, changes, or walks back
